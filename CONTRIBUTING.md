# Contributing to Dynamic Weather Information System

Thank you for your interest in contributing to the Weather App project! We welcome contributions from the community.

## How to Contribute

1. **Fork the repository** and clone it locally
2. **Create a new branch** for your feature or bug fix: `git checkout -b feature/your-feature-name`
3. **Make your changes** and test thoroughly
4. **Commit your changes** with clear, descriptive messages
5. **Push to your branch** and create a Pull Request

## Development Setup

### Prerequisites
- Modern web browser (Chrome, Firefox, Safari, Edge)
- Code editor (VS Code, Sublime, etc.)
- Git
- OpenWeatherMap API key (free tier available)

### Installation
```bash
# Clone the repository
git clone https://github.com/abhilashbha/weather-app.git
cd weather-app

# Update API key in the JavaScript files
# Get your free API key from: https://openweathermap.org/api

# Start a local server (Python)
python -m http.server 8000

# Or use Node.js
npx http-server

# Open browser and navigate to: http://localhost:8000
```

## Guidelines

- Follow the existing code style and conventions
- Test features across multiple browsers
- Ensure responsive design is maintained
- Document any new features or changes
- Write clear commit messages
- Test with valid API keys

## Enhancement Ideas

- Add weather alerts/notifications
- Implement dark mode
- Add more chart visualizations
- Improve mobile responsiveness
- Add weather comparison between cities
- Implement caching strategies

## Bug Reports and Feature Requests

Please open an issue with:
- Clear description of the bug or feature
- Steps to reproduce (for bugs)
- Expected vs actual behavior
- Browser and OS details
- Screenshots if applicable

## Questions?

Feel free to open an issue or contact the maintainers.

Thank you for contributing! 🎉
